Featured Highlights

Three security directions reshaping the field

These are among the most important shifts in AI security today: generative interaction, edge implementation realism, and real-world physical intelligence.

Generative AI

Prompt-layer security is now a system security problem

Generative AI security is not only about harmful outputs. It includes prompt injection, jailbreaks, tool abuse, misuse, alignment failure, model exposure, and secure containment across real applications.

Edge AI

At the edge, hardware details become part of security

Physical access, leakage, timing behavior, firmware trust, model extraction, secure update paths, and accelerator reality change the attacker model completely.

Physical AI

Trust must cover sensing, computation, timing, and action together

In physical AI, cyber decisions influence real-world behavior, so security, safety, latency, control, and reliability must be studied together.

Research Perspective

Cross-layer security with a hardware-aware lens

My work sits at the boundary of AI security, hardware security, edge intelligence, and trustworthy deployment. A central theme is that real security cannot stop at algorithmic robustness alone. It must also account for implementation, accelerators, physical access, timing behavior, system integration, and deployment constraints.

  • Security across software, hardware, and deployment layers
  • Side-channel and fault-injection views for edge AI
  • Trust, resilience, and countermeasures for physical AI
  • Structured synthesis of open gaps and future directions
