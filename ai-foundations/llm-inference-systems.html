LLM Inference Systems

Large language model inference is not just a model problem. It is also a serving problem shaped by tokenizer flow, prefill versus decode behavior, KV-cache growth, memory bandwidth limits, batching strategy, request scheduling, and the tension between latency and throughput. These factors strongly influence security and observability.

Overview

What this topic covers

Transformer inference has at least two different system phases. Prefill processes prompt context with high parallelism, while decode often becomes more latency-sensitive and incrementally memory-bound as the model generates tokens. KV-cache management becomes central because intermediate attention state must remain accessible across steps.

This system view matters for security because prompt length, batching choice, generation dynamics, and cache residency can all affect timing, utilization, and memory traffic. In multi-user or cloud settings, these behaviors can intersect with privacy, co-tenancy, isolation, and service-level abuse.

Why it matters

Research significance

  • LLM serving behavior is shaped heavily by cache growth, token generation dynamics, and scheduling policy.
  • Latency bottlenecks and throughput bottlenecks expose different system-level signals and control points.
  • Serving systems create new security boundaries involving request multiplexing, resource sharing, and prompt handling.
  • Inference realism is essential for studying extraction, leakage, and availability threats in LLM deployment.

Discussion

Key technical questions

A model-only view of LLM security misses many of the questions that arise once serving enters the picture.

Threat model

Is the attacker a user of the service, a co-tenant, an observer of response timing, or someone with system-level access to logs, memory, or scheduling metadata?

System constraints

Batch size, prompt length, decoding strategy, caching policy, and latency targets all shape what is observable or exploitable in a real inference service.

Open direction

An important research direction is to tie LLM serving metrics—such as time-to-first-token, token throughput, and cache residency—to clearer security models for privacy, extraction resistance, and operational containment.
